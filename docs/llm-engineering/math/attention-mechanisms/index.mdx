# Attention Mechanisms

Attention is the algorithm that made transformers possible — and transformers made modern LLMs possible. At its core, attention answers a simple question: given a sequence of tokens, which ones should the model "pay attention to" when processing each position? The answer to that question is learned during training, and the mechanism for computing it is elegant, parallelizable, and remarkably effective.

## The Intuition

Consider the sentence: **"The cat sat on the mat because it was tired."**

What does "it" refer to? The cat or the mat? A human reader uses context to resolve this ambiguity. Attention mechanisms do the same thing mathematically: they compute relevance scores between every pair of tokens, allowing the model to learn that "it" should attend strongly to "cat" (biological entities get tired, mats don't).

## Self-Attention: The Core Algorithm

Self-attention lets each token in a sequence look at every other token to compute a context-aware representation. Here's how it works, step by step:

### Step 1: Create Query, Key, and Value Vectors
For each token, the model creates three vectors by multiplying the token's embedding by learned weight matrices `W_Q`, `W_K`, and `W_V`:

- **Query (Q)**: "What am I looking for?" — represents what this token wants to know about other tokens.
- **Key (K)**: "What do I have?" — represents what information this token can provide to others.
- **Value (V)**: "What should I communicate?" — the actual information this token contributes.

For a sequence of length n with embedding dimension d, Q, K, and V are each n × d matrices.

### Step 2: Compute Attention Scores
Attention scores = Q × K^T (matrix multiplication)

This produces an n × n matrix where entry (i, j) represents how much token i should attend to token j. Higher scores mean more relevance.

### Step 3: Scale and Normalize
Scale the scores by `√d_k` (where `d_k` is the key dimension) to prevent the dot products from growing too large and pushing the softmax into regions of extremely small gradients.

Then apply **softmax** along each row — converting raw scores into a probability distribution over all tokens: attention_weights = softmax(QK^T / √d_k)

### Step 4: Weighted Sum
Multiply attention weights by the value matrix V. The result: each token's new representation is a weighted sum of all tokens' values, where weights represent relevance: output = attention_weights × V

## Multi-Head Attention

Single-head attention captures one type of relationship. Multi-head attention runs multiple attention operations in parallel, each with its own `W_Q`, `W_K`, `W_V` matrices, learning different types of relationships:

- One head might learn syntactic relationships (subject-verb agreement).
- Another might learn semantic relationships (synonyms, antonyms).
- Another might learn positional relationships (nearby tokens matter more).
- Another might learn factual relationships (entity co-references).

The outputs from all heads are concatenated and projected back to the model dimension. Multi-head attention is what gives transformers their remarkable representational power.

### Causal Masking (Autoregressive)
In decoder-only models like GPT, each token can only attend to tokens that came before it (causal attention). This is enforced by a triangular mask that sets attention scores to -∞ for future positions before the softmax. This ensures the model generates text sequentially and doesn't "cheat" by looking ahead.

## Efficiency: The O(n²) Problem

Full attention computes n × n scores for every layer. For n = 100,000 (a 100K token context window), that's 10 billion scores — per layer, per head. This is the fundamental scalability bottleneck of transformers.

### Solutions

**Flash Attention** — The breakthrough that made long contexts practical. Instead of materializing the full n × n attention matrix in GPU high-bandwidth memory (HBM), flash attention computes attention in tiles (blocks), keeping intermediate results in faster on-chip SRAM. It's mathematically identical to standard attention but up to 7x faster and uses linearly less memory. Flash Attention 2 and 3 continue to push these optimizations further.

**Sparse Attention** — Most attention weights are near zero. Sparse attention exploits this by only computing attention for a subset of token pairs: sliding window (each token attends to its k neighbors), dilated sliding window (skip n tokens between attended positions), global attention (a few tokens attend to everything and everything attends to them).

**Grouped-Query Attention (GQA)** — Uses fewer key-value heads than query heads. Multiple query heads share the same key-value pair. Llama 3 uses GQA with 8 KV heads and 32 query heads — reducing KV cache size by 4x with minimal quality loss.

## Why Attention Mechanisms Are Worth Understanding Deeply

Attention isn't just an implementation detail — it determines what your model can learn and what it costs to run. Long-context performance, multi-lingual capability, instruction following, hallucination rates — all trace back to properties of the attention mechanism. When you understand attention, you understand the fundamental capabilities and limitations of transformer-based LLMs.
